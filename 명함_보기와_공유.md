# 디지털 명함 – 보기 & 공유 방법

---

## 로컬에서 먼저 확인하기 (추천 순서)

1. **프로젝트 폴더 열기**  
   `바이브코딩` 폴더로 이동합니다.

2. **이미지 폴더·파일 확인**  
   - 폴더 이름: **`image`** (또는 `이미지` 둘 다 동작)  
   - 프로필 사진 파일명: **`profile.jpg`** 또는 **`profile.png`**  
   - 경로 예: `바이브코딩/image/profile.jpg`

3. **터미널에서 서버 실행**  
   `바이브코딩` 폴더에서 아래 명령을 실행합니다.
   ```powershell
   python -m http.server 5500
   ```

4. **브라우저에서 접속**  
   주소창에 **http://localhost:5500** 입력 후 엔터.

5. **확인**  
   - 명함 카드에 프로필 사진이 보이는지  
   - 「명함 확인」 클릭 시 소개 페이지로 넘어가는지  
   - 휴대전화·이메일 링크가 잘 동작하는지  

서버를 끄려면 터미널에서 **Ctrl + C** 를 누르면 됩니다.

## 1. 로컬에서 보기 (다른 방법)

### 방법 A: 간이 웹 서버 (권장)
프로젝트 폴더에서 터미널을 열고:

```powershell
python -m http.server 5500
```

브라우저에서 **http://localhost:5500** 접속  
→ 명함 카드가 보이고, 클릭 시 소개 페이지로 이동합니다.

### 방법 B: 파일 더블클릭
`index.html` 파일을 더블클릭해서 열어도 됩니다.  
(같은 폴더에 `profile.html`, `style.css`, **`image`** 폴더가 있으면 링크가 그대로 동작합니다.)

### 방법 C: blog.py와 함께 보기 (시연 허브 + 명함)
```powershell
python blog.py
```

- **시연 문의 허브**: http://127.0.0.1:8000  
- **디지털 명함**: http://127.0.0.1:8000/card  

한 번에 두 페이지 모두 사용할 수 있습니다.

## 2. 공유하기 (URL로 나눠 주기)

다른 사람에게 **링크 하나**만 보내면, 그 주소로 들어가 명함을 보고 소개 페이지까지 볼 수 있어야 합니다.  
그러려면 **인터넷에 배포**해야 합니다.

### 옵션 1: 지금 쓰는 Render 배포 활용 (blog.py)
이미 **Render**에 배포 중이라면:

1. 코드 수정 후 **beta** 브랜치에 push  
   ```powershell
   .\upload.ps1 -Message "feat: 디지털 명함 페이지 추가"
   ```
2. Render 배포가 끝나면, 아래 주소로 명함 공유:
   - **Beta**: `https://drpalette-beta.onrender.com/card`
   - **운영(Prod)**: `https://drpalette-prod.onrender.com/card` (main 반영 후)

→ **공유할 링크**: `https://드팔레트서비스주소/card`  
(실제 Render 대시보드에 나오는 URL로 바꿔서 사용하세요.)

### 옵션 2: GitHub Pages / Netlify / Vercel (정적 페이지만)
명함만 따로 공유하고 싶다면:

- **GitHub Pages**: 이 저장소를 올린 뒤 Settings → Pages에서 브랜치 선택 후 배포.  
  → `https://<사용자명>.github.io/<저장소명>/` 로 접속 가능.
- **Netlify / Vercel**: 프로젝트 폴더를 연결한 뒤 배포하면 고유 URL이 생깁니다.  
  → 그 URL을 명함 링크로 공유하면 됩니다.

## 요약

| 목적       | 방법                          | 주소 예시                    |
|------------|-------------------------------|------------------------------|
| 내 PC에서 보기 | `python -m http.server 5500`  | http://localhost:5500        |
| 서버에서 보기   | `python blog.py`              | http://127.0.0.1:8000/card  |
| 다른 사람에게 공유 | Render 등에 배포 후 URL 공유 | https://xxx.onrender.com/card |

공유할 때는 **명함 첫 화면 주소**만 알려주면 됩니다.  
그 주소로 들어가서 카드를 클릭하면 소개 페이지가 열리도록 되어 있습니다.

## 3. 프로필 사진 (이미지 폴더) — 사진이 안 보일 때

- **폴더 위치**: `바이브코딩` **안에** **`image`** 폴더가 있어야 합니다.  
  (프로젝트에 `image` 폴더를 만들어 두었으니, 그 안에 사진만 넣으면 됩니다.)
- **파일 이름**: 반드시 **`profile.jpg`** 또는 **`profile.png`** 로 저장해 주세요.  
  (다른 이름이면 브라우저가 찾지 못합니다.)
- **경로 확인**: `바이브코딩/image/profile.jpg` 또는 `바이브코딩/image/profile.png`  
  → 이 경로에 파일이 실제로 있는지 탐색기에서 확인해 보세요.
- **로컬 확인 방법**:  
  1. `바이브코딩` 폴더에서 `python -m http.server 5500` 실행  
  2. 브라우저에서 **http://localhost:5500** 접속 (주소창에 직접 입력)  
  3. **http://localhost:5500** 으로 열었을 때만 이미지 경로가 맞게 동작합니다.  
     (다른 폴더에서 연다거나, `file://` 로 열면 경로가 틀어질 수 있습니다.)

## 4. 카카오톡 등에 링크 보낼 때 미리보기 (썸네일)

링크를 보내면 **“강명구 · 디지털 명함”** 제목과 설명, 사진이 미리보기로 나오게 하려면:

1. **배포가 된 상태**에서, 명함 페이지 주소(예: `https://xxx.onrender.com/card`)가 있어야 합니다.
2. **index.html**의 `<meta property="og:image" ...>` 에 **절대 주소**를 넣어 주세요.  
   예: `https://도메인/card/image/profile.jpg`  
   (실제 배포 주소에 맞게 수정)
3. 그 다음부터 같은 링크를 카카오톡 등에 보내면, 제목·설명·이미지가 미리보기로 뜹니다.

## 5. 나중에 사진이 여러 장일 때 (요청 예시)

사진을 여러 장 쓰고 싶을 때는 아래처럼 요청하시면 됩니다.

| 하고 싶은 것 | 요청 예시 |
|-------------|-----------|
| 명함에 프로필 1장 + 팀/제품 사진 1장 넣기 | "명함 카드에 프로필 사진 아래에 팀 사진도 한 장 넣어줘" |
| 소개 페이지에 사진 여러 장 갤러리로 보이게 | "profile 페이지에 이미지 폴더에 넣은 사진들 갤러리로 보이게 해줘" |
| 특정 위치에 특정 이미지 지정 | "image 폴더에 team.jpg 넣었어. 소개 문구 아래에 team.jpg 보이게 해줘" |
| 슬라이드/캐러셀로 보이게 | "소개 페이지에 사진 3장 넣어서 좌우로 넘기는 슬라이드로 해줘" |

**요청할 때 알려 주시면 좋은 것**  
- 폴더 이름 (예: `image`)  
- 파일 이름 (예: `profile.jpg`, `team.png`)  
- 어디에 어떻게 보이게 할지 (명함 상단, 소개 페이지 갤러리, 슬라이드 등)
